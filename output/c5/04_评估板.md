# 评估板

如果你仍处于开发过程的早期阶段，或者只是评估 Android，你可能想依靠一块评估板。以下是你在选择时可能需要考虑的一些因素：

**SoC** — 它是否依赖你最终设计中使用的 SoC？它是同一系列的 SoC 吗？或者它是你计划使用的尚未发布的 SoC 的先前版本？

**社区** — 板周围是否有社区，或者制造商是唯一支持来源？这个社区有多活跃？它是围绕单个板还是一系列板建立的？

**成本** — 板的预付成本是多少，包含什么？附加组件或扩展的成本是多少？低端选项和高端选项之间的价格差异是多少，有什么功能差异？

**功能** — 板包含/暴露了哪些功能？SoC 越来越多地支持广泛的功能。然而，板使越多 SoC 功能可用，它往往就越贵。你正在看的板是否暴露了你需要的功能？

**可扩展性** — 板提供的基本功能可能足以完成你试图完成的一定百分比的事情，但板是否允许你连接额外的硬件，以便你可以模拟你正在努力实现的最终功能？

**可用性** — 多容易获得这个板？有些板在纸面上看起来很好，但供应不稳定。

**许可** — 你可以按原样使用该板用于最终产品吗？有些制造商禁止你这样做。你能拿到物料清单（BOM）和原理图吗？

**目录零件** — 板使用的是目录零件吗？如果板依赖非目录零件，你将需要去找它们的制造商来获得它们。

**第三方零件** — 有时 SoC 供应商在其板中包含第三方零件。确保你对这些组件应用类似的标准。

**软件支持** — Android 在板上的支持情况如何？由谁支持？制造商？第三方？支持哪些版本的 Android？
